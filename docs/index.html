My Website

This is my website.

This document is created using Markdown.

I created my-website repository on irsbugs account of github

In creating i I didn't add any README.md so there were no files in the account.

I then went to my laptop where a mkdocs site folder has had git applied.

I issued the command:

$ git remote add origin https://github.com/irsbugs/my-website.git

$ git push -u origin master

After this went to my github account and the files and folders from my laptops site folder were in
https://github.com/irsbugs/my-website/...

In github Settins I set the Github Pages source to be master branch. I was then able to use my laptops browser to go to https://irsbugs.github.io/my-website/index.html and the web-site was available.

For some reason https://irsbugs.github.io/my-website/ would give a 404 error but it was OK with the index.html appended to it.

Example of updating after editing index.md

Update mkdocs and check in local browser

(venv7) ian@X200:~/my-website$ mkdocs build --clean && mkdocs serve

[I 190828 07:25:24 server:296] Serving on http://127.0.0.1:8000
[I 190828 07:25:24 handlers:62] Start watching changes

Now commit the updates in site folder

(venv7) ian@X200:~/my-website$ cd site
(venv7) ian@X200:~/my-website/site$ git commit -a -m "updates to index.md"
 4 files changed, 14 insertions(+), 4 deletions(-)
(venv7) ian@X200:~/my-website/site$

Note: If you leave out the -m "message" then a session of nano will open to prompt you to add the message.
